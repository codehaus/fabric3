The themes of the 0.3 release are:

  • Usability and Application API Stability
  • JPA and Hibernate integration
  • Resource injection
  • Improved transaction management
  • Improved SCA 1.0 spec compliance
  • New and Improved Extensions

New features are described below.

Usability

The 0.3 release introduces numerous bug fixes and stabilizes the end-user application API. From this release on, Fabric3 will focus on maintaining backward compatibility for applications written against the 0.3 APIs. In addition, 0.3 includes improved monitoring and error reporting.

The 0.3 release also adds support for mocking SCA components using EasyMock. Combined with the existing JUnit support, this simplifies automated integration testing of SCA applications by allowing their interactions with other components to be easily mocked and verified.

Native JPA and Hibernate Integration

The 0.3 release also introduces integration with Hibernate and JPA implementations for persisting data. This allows SCA components to directly access information stored in relational databases using the standard JPA APIs. Support is fully integrated with SCA declarative transaction management.

Resource Injection

The 0.3 release supports the use of @Resource injection for accessing system-supplied resources such as DataSources.

Improved Transaction Management

The 0.3 release includes transactional integration with JPA, Hibernate and system-supplied DataSources.

SCA Compliance

The 0.3 runtime includes support for complex properties, setting of properties via external files, XPath evaluation, and support for composite service and reference promotion. The 0.3 runtime also provides support for service and reference bindings contained directly in component definitions.

New and Improved Extensions

New fabric3-axis2 extension supporting Web Services using Apache Axis2.

New fabric3-jpa-hibernate extension supporting Hibernate.

New fabric3-mock extension for testing SCA components and composites with EasyMock.

New fabric3-resource extension supporting JSR-250 and fabric3 @Resource annotations.
