The themes of the 0.2 release are:

  • Support for SCA policy intents and definitions
  • Support for declarative transactions
  • Enhanced contribution and extension functionality
  • Improved SCA spec compliance
  • Further modularization

New features are described below.

SCA Policy Support

The 0.2 release introduces support for SCA policy intents and definitions. Policy definitions may be contributed to a domain and activated using the contribution service. Architecturally, SCA policy support is provided by an extensible policy framework.

Declarative Transactions

The 0.2 release also introduces support for declarative transactions, including 2-phase commit. Integration with JEE application server transaction managers (WebLogic Server, JBoss, Geronimo) and standalone transaction managers (JOTM) is provided.

Contribution Handling

Contribution handling has been significantly improved, including:

  • Support for self-contained archives with third-party libraries (jars) packaged in META-INF/lib
  • A new Maven-based contribution plugin with support for packaging transitive dependencies
  • Support for SCA contribution manifests and extensible import/export

Extension Handling

Runtime extensions (e.g. bindings, component implementation types) are now be packaged as SCA archives and installed as follows:

  • In the standalone runtimes, they are added to an extensions directory
  • In the web app runtime they are bundled in the WEB-INF/fabric3/extensions directory of the WAR
  • In the development runtime, they are added to an extensions directory
  • In the Maven runtime, they are specified in the iTest plugin configuration and automatically downloaded from a repository

The need to modify system composites to add extensions has been eliminated. Also, since runtime extensions are SCA contributions, they may be self-contained and include third-party libraries in META-INF/lib.

SCA Compliance

Specification support has also been improved, including the following:

  • Support for references with multiplicity greater than 1
  • Improved web application support, including web app-specific component types.
  • Autowire include and inheritance support
  • Fixes for various wire resolution and promotion bugs.

Runtime Modularization

  • Separate single-VM, standalone, and participant distributions have been created.
  • Various modules, including the console, Jetty HTTP service, and contribution scanner have been converted into extensions and are no longer included in the core system composite.
  • Runtime implementation classes have been isolated from the extension classpath.
