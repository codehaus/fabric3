Loading and configuring the IDEA project

In IDEA, select File..Open Project and choose tutorials.ipr located in the directory where the Fabric3 tutorial distribution was extracted. This will open the tutorials project which will be used as you proceed through the tutorials. The indivudal tutorials are divided into a number of modules. The fabric3-tutorial-loanapp-common module contains shared artifacts. In this tutorial, we will be working with the fabric3-tutorial-loanapp-simple module.

Before we begin, the project must include the SCA and runtime API jars included with the development runtime distribution. In the Project pane to the left, right click on the fabric3-tutorial-loanapp-simple module and select 'Module Settings...'. the following window should appear:

You will need to create a Global Library corresponding to the fabric3-dev-runtime dependency highlighted by (1). Select "Add.." (2) and create the Global Library containing the following jars which are located in the /lib directory of the Fabric3 Runtime Distribution:

  • development-api-0.2.jar
  • fabric3-api-0.2.jar
  • fabric3-host-api-0.2.jar
  • sca-api-r1.0-0.2.1.jar
  • geronimo-annotation_1.0_spec-1.1.jar (JSR-250 Commons Annotations definitions)

You will also need to create a Global Library corresponding to the jaxb-api-2.1 entry. If you have build the tutorials using Maven, this library will have been downloaded to your local Maven repository. You can point the Global Library to the Maven repository archive.

The project will now be configured to use the development runtime. When creating new projects, you can directly point to the Global Library that was created in this step, thereby avoiding the need to configure the runtime for every project.
