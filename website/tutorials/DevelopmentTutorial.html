Overview

This tutorial covers the following:

  • Setting up a development environment with Fabric3
  • Creating a composite with a LoanApplicationService
  • Initializing a local SCA domain and activating the composite in it
  • Connecting to the local domain and invoking the LoanApplicationService

The Fabric3 Development Runtime

The Fabric3 Development Runtime (F3DR) provides a rapid prototyping container that integrates with popular IDEs, including Eclipse and IDEA IntelliJ. It allows developers to create and run SCA composites without having to set up a complex server environment. As the runtime is based on a small-footprint design, startup time should be in the sub-second range.

Please note that F3DR is not intended for testing purposes. Unit testing SCA composites does not require a container; instead instantiate component implementation classes using a test harness such as JUnit or TestNG and set component references with mock objects. We recommend using EasyMock and SCAMock to do this.

To perform integration testing, please see the [Fabric3 iTest Runtime], which offers a complete automated SCA testing framework with Maven.

Getting Started

Perform the following steps:

  1. Download and extract the Fabric3 Development Runtime Distribution from the Downloads Page
  2. Download and extract the Tutorials Distribution from the Downloads Page

The tutorial distribution includes both Eclipse and IDEA project files. If you are using IDEA, please proceed here. If you are using Eclipse, please proceed [here].
