Abstract representation of SCA artifacts independent of implementation.

Explanation

Fabric3 makes the distinction between logical and physical code and configuration. Every artifact contributed to the Fabric3 runtime contains SCA metadata, such as a sca-contribution.xml, componentType.xml, .composite, etc. Fabric3 loads these definitions into the runtime as logical models. These local models store all the SCA metadata in memory and allow the controller to intelligently manage the SCA contribution according to the SCA specification. Fabric3 uses SCDL4J and fabric3-scdl to model SCA metadata as Java objects.
