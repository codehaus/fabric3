Declares the access mechanism used to invoke a service or the access mechanism a reference uses to invoke a service.

Explanation

A SCA binding is similar to a WSDL binding in that it specifies a particular transport protocol and message format for accessing a service. A binding is associated with a service or reference declaration and defines what transport protocol and configuration is used to access the service.

Example

sample.composite
<?xml version="1.0" encoding="UTF-8"?>
<composite xmlns="http://www.osoa.org/xmlns/sca/1.0"

           xmlns:burlap="http://www.fabric3.org/binding/burlap/0.2"
           name="BurlapTestComposite">

    <service name="HelloBurlapService" promote="HelloService">

        <interface.java interface="org.fabric3.sample.hello.HelloService"/>
        <burlap:binding.burlap uri="/helloService" />
    </service>

    <reference name="HelloBurlapReference" promote="HelloClient/helloService">

        <interface.java interface="org.fabric3.sample.hello.HelloService"/>
        <burlap:binding.burlap uri="http://localhost:8900/helloService"/>
    </reference>

    <!-- Hello Client -->

    <component name="HelloClient">
        <implementation.java class="org.fabric3.sample.hello.HelloClientImpl"/>
		  <reference name="helloService" target="HelloService"/>

    </component>

    <!-- Hello Service -->
    <component name="HelloService">
        <implementation.java class="org.fabric3.sample.hello.HelloServiceImpl"/>
    </component>

</composite>
Reference

For illustrative purposes the reference is resolved from a binding but since both service and reference are in the same composite the reference target attribute would be more practical to use.

<reference name="helloService" target="HelloService"/>

Notice in this example that the HelloService service is bound to the burlap server at /helloService and then the reference accesses this service at http://localhost:8900/helloService and provides itself to the HelloClient reference "helloService"
