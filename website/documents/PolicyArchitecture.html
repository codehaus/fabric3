Introduction

This document explains the F3 implementation of the SCA policy specification. SCA policy specification describes a framework for enforcing QoS aspects of the runtime behaviour of components and the interaction between components using abstract requirement assertion called intents and more concrete mechanisms called policies to which those intents map. Typical QoS aspects that would be addressed by the SCA policy framework would include security, reliability, transactionality, auditing, tracing etc. The SCA policy specification can be found at here.

Core Functional Concepts

SCA Policy framework allow end users to specify abstract assertions against the runtime behaviour of components as well as interaction between components using services and references. These assertions are called Intents in the SCA policy speak. Intents can be defined against any SCA artifact within the SCDL, however, they always apply to either bindings or implementations. Intents applicable to bindings are called interaction intents and those for implementations are called implementation intents.
